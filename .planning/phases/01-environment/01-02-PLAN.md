   ```python
   print("=" * 50)
   print("  Evo-RL Environment Verification")

   # 1. CUDA
   print("[1/3] CUDA")
   import torch
   cuda_ok = torch.cuda.is_available()
   if cuda_ok:

       gpu_name = torch.cuda.get_device_name(0)

       print(f"  GPU 0: {gpu_name} ({gpu_mem:.1f} GB)")
       print(f"  PyTorch: {torch.__version__}")
   else:
       print("  Status: NOT AVAILABLE")
       all_pass = False
   print()
   ```

   ```python
   print("[2/3] Dependencies")
   deps = [
       ("transformers", None),
       ("accelerate", None),
       ("datasets", None),
       ("lerobot", None),
       ("wandb", None),
       ("huggingface_hub", None),
   ]
   for pkg_name, _ in deps:
       try:
           mod = __import__(pkg_name)
           ver = getattr(mod, "__version__", "unknown")
           print(f"  {pkg_name}: {ver}")
       except ImportError:
           print(f"  {pkg_name}: MISSING")
           all_pass = False
   print()
   ```

6. **Section 3 - Pretrained weights** (path existence only, no loading):
   ```python
   print("[3/3] Pretrained Weights")
   weights = {
       "pi05_base": "lerobot/pi05_base",
       "siglip-so400m-patch14-384": "google/siglip-so400m-patch14-384",
       "gemma-3-270m": "google/gemma-3-270m",
       "paligemma-3b-pt-224": "google/paligemma-3b-pt-224 (tokenizer)",
   }
   pretrained_abs = os.path.abspath(PRETRAINED_DIR)
   for dirname, hf_id in weights.items():
       path = os.path.join(pretrained_abs, dirname)
       found = os.path.isdir(path) and len(os.listdir(path)) > 0
       status = "OK" if found else "MISSING"
       if not found and dirname != "paligemma-3b-pt-224":
           all_pass = False
       print(f"  {dirname}: {status}")
       if not found:
           print(f"    HF ID: {hf_id}")
   print(f"  Path: {pretrained_abs}")
   print()
   ```
   Note: paligemma-3b-pt-224 is optional (tokenizer already present on most setups), so its absence does NOT fail the check.

**Structure** (sections in this exact order):

1. **Title and overview**:
   ```markdown
   # Evo-RL Environment Setup Guide

   This guide covers setting up the `evo-rl` conda environment on supported hardware
   (RTX 4090 local machine, A100 server). After following this guide, run `verify_env.py`
   to confirm everything is ready.
   ```

2. **Prerequisites**:
   - conda (Miniconda or Anaconda)
   - NVIDIA GPU with CUDA support
   - `huggingface-cli` (installed as part of `huggingface-hub`)
   - SSH access to A100 (for remote setup only)

3. **Section: Local Machine Setup (RTX 4090)**:

   3a. Create conda env:
   ```bash
   conda create -y -n evo-rl python=3.10
   conda activate evo-rl
   ```

   3b. Install Evo-RL (editable mode per D-05):
   ```bash
   cd /path/to/Evo-RL
   pip install -e .
   ```

   3c. Note about optional extras: transformers is needed for policy training:
   ```bash
   pip install -e ".[transformers-dep]"
   ```

   3d. Verify installation:
   ```bash
   python scripts/verify_env.py
   ```

4. **Section: Pretrained Weights**:

   4a. Download all weights:
   ```bash
   bash scripts/download_weights.sh
   ```

   4b. Expected directory structure after download (from RESEARCH.md weight storage convention):
   ```
   Evo-RL/pretrained/
   ├── pi05_base/              (~14GB, lerobot/pi05_base)

   ├── gemma-3-270m/           (~0.5GB, google/gemma-3-270m)
   └── paligemma-3b-pt-224/    (tokenizer, already present)
   ```

   4c. HF mirror note (per D-11): If HuggingFace is unreachable, set `HF_ENDPOINT`:
   ```bash
   export HF_ENDPOINT=https://hf-mirror.com
   bash scripts/download_weights.sh
   ```

   4d. Tokenizer path fix for offline machines (from RESEARCH.md Pitfall 3):
   After downloading `pi05_base`, if running on a machine without HF network access, edit `pretrained/pi05_base/policy_preprocessor.json` and change the tokenizer reference from `google/paligemma-3b-pt-224` to the local relative path `../paligemma-3b-pt-224`. See `docs/reproduction_guide.md` for details.
